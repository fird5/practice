What You'll Learn

When students ask AI a complex question and get a vague or incorrect answer, they blame the AI. The real problem is almost always the question itself. AI processes exactly what you give it — if the input is unstructured, the output will be too.

Computational Logic is the discipline of decomposing any problem into a step-by-step decision tree before prompting. It is how engineers, data scientists, and strategists think — and it is the single skill that separates average AI outputs from exceptional ones.

What You'll Learn

You will learn Dr. Priya's SCOPE framework — a 5-step method for mapping any academic or business problem into a structured logic tree before prompting. SCOPE turns vague questions into precise, AI-solvable sequences.

Applied to NotebookLM: you will use your logic tree to query your Notebook with surgical precision — each node in the tree becomes a separate, targeted prompt that builds toward a complete, cited answer.

The SCOPE Framework
🔵
S — Situation. Define the full context. Who is involved? What domain? What constraints apply? Give AI the complete situational picture before asking anything.
🎯
C — Core Question. State the single most important question you need answered. One question per logic branch — never combine multiple questions in one prompt.
⚙️
O — Output Format. Specify exactly what structure you want. "A numbered list of 5 items" beats "explain this." Format precision dramatically improves AI output quality.
📏
P — Parameters. Set explicit rules. Word count, tone, perspective, what to exclude. Parameters prevent AI from drifting into generic, unhelpful responses.
✅
E — Evidence Required. Specify whether citations, data, or examples are required. For academic work, always demand source citations. NotebookLM will link to your uploaded documents.

What You'll Learn

A live over-the-shoulder demo applying the full SCOPE framework to a real cross-border logistics problem in SEA — from raw problem statement to a structured NotebookLM decision tree to a final, actionable output.

You will see how a problem that would produce a vague, 500-word AI response when asked directly produces a precise, multi-part cited analysis when run through a logic tree. This is the difference between using AI and commanding it.

What to Avoid
🚫
Don't skip the logic tree step. It feels slower at first. It is not. A 5-minute tree setup saves 30 minutes of iterating on bad outputs. Do the thinking before the prompting.
🚫
Don't combine multiple SCOPE steps into one prompt. Each node in your tree is a separate prompt. Mixing Situation + Core Question + Output Format into one query collapses the structure and loses precision.
🚫
Don't accept the first output. Run your logic tree prompt, then ask: "What did you assume that I didn't state explicitly?" The AI's answer to that question reveals the gaps in your SCOPE setup.

Sprint Tips from Dr. PriyaRead before you begin

Start with the Situation step — fully map the company's context before building any decision branches. Students who skip this produce trees that assume facts not in evidence.
🎯
Your decision tree quality is the score, not the writing quality. A clear, logical tree with 6 well-sourced decision nodes outscores a beautifully written paragraph with no structure.
